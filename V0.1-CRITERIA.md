## Cut line (one sentence)

wraith v0.1 **detects and CONFIRMS** SSRF and emits suite-standard findings plus read-only recon. It does **NOT** weaponize: no code execution, no target state-change, no use of harvested credentials, no reverse shell, no pivot. Detect+confirm is where the durable value and the SSRFmap-replacement differentiation live; weaponization is a v0.2 module behind an explicit, sandboxed `--exploit` gate.

## v0.1 acceptance criteria (testable)

1. **Input flexibility.** Accept a raw HTTP request file (SSRFmap parity) **and** CLI flags. The injection point (query param / header / body field / path segment) is explicitly markable.
2. **Filter-bypass mutator engine.** Given a seed (intended-trusted host) + an internal target, generate the variant catalog: IP encodings (dword-decimal, hex dotted + dotless, octal dotted + dotless, shorthand `127.1`/`0`, IPv4-mapped IPv6 `[::ffff:…]`, IPv6 loopback/link-local/ULA, `0.0.0.0`), userinfo `@` (incl. multi-`@`), fragment `#`, backslash `\`, whitespace/tab, CRLF, malformed/long scheme. **Each mutator unit-tested against exact-byte expected vectors** (see Testability Tier 0). Default mutator ordering: redirect-chain + `@` → DNS-rebind (OOB) → stack-aware parser-differential → encoding.
3. **Cloud-metadata detection probes.** AWS IMDSv1 GET **and** IMDSv2 PUT-token→GET handshake (**including the header-injection variant** where the SSRF primitive controls request headers — cf. Typebot CVE-2025-64709), GCP (`Metadata-Flavor: Google`), Azure (`Metadata: true` + `api-version`, omit `X-Forwarded-For`), Alibaba (`100.100.100.200`), Oracle (`192.0.0.192`), DigitalOcean (`169.254.169.254/metadata/v1.json`). Detect credential-shaped responses and **emit a finding**. MUST NOT auto-use harvested credentials.
4. **OOB confirmation engine (the primary gap over SSRFmap — mandatory).** interactsh-compatible client (+ self-host config). Unique token per probe, DNS **and** HTTP capture, polling with a persistent late-hit window. **A DNS-only callback counts as CONFIRMED** (finding flagged "HTTP egress likely filtered"). Blind SSRF is the default case; every payload is paired with an OOB canary.
5. **Protocol modules — read-only + generator only.** `dict://` read-only recon (port/banner, Redis `INFO`, Memcached `stats`). `gopher://` **payload generator** (RESP + FastCGI byte encoders, correct `%0d%0a`, single/double URL-encode toggle) — **dry-run by default, behind an explicit opt-in flag, emits the payload for the operator; does NOT fire weaponized sequences.**
6. **MCP / AI-infra detection catalog (differentiator).** 5 curated, **version-gated** detection signatures reusing the injection + OOB + finding-schema machinery (not a bespoke engine): Fetch MCP (CVE-2025-65513), Microsoft MarkItDown MCP, MCP-Atlassian (CVE-2026-27826), LiteLLM `/v1/rag/ingest` `file_url`, LangChain `RecursiveUrlLoader` (CVE-2026-26019 / -27795). Version-gate and cite; some are unpatched/2026-dated.
7. **Async/concurrent execution.**
8. **Output.** Every request routed through the **`scan-primitives`** shared client (scope-enforced, rate-limited, proxy-aware — **build that lib first**). Every result emitted as a suite-standard finding (target, injection vector, payload variant, response signature, OOB proof, severity, timestamps) + SARIF 2.1.0 export + HackerOne markdown via `h1-reporter`.
9. **Hermetic test suite** (see Testability) + the suite `ship_gate` wheel test.

## Explicitly NOT in v0.1 (deferred)

- **Weaponized `gopher://` exploitation** (Redis cron/SSH/`MODULE LOAD`, FastCGI php-fpm RCE) → v0.2 behind `--exploit`/`--dangerous`, sandboxed. Destructive (`FLUSHALL`) + narrow landing conditions.
- **Using harvested cloud credentials** (SDK calls to enumerate/exfil) — detection emits the finding; cred-use is a separate dangerous pivot.
- **Reverse-shell / connect-back listener** (SSRFmap had this) — weaponization.
- **Request-smuggling desync** to reach enforced IMDSv2 — that's `doppelganger`'s problem.
- **Self-hosted DNS-rebinding orchestration** — v0.1 may *emit* rebind hostnames pointing at public rebind services (rbndr.us / 1u.ms) as mutators; hosting the flip is deferred.
- **Full CVE template library** (Ivanti/GitLab/Spring/Next.js/Grafana as maintained templates) — seed a handful, defer the catalog.
- Extended protocol weaponization (MySQL/Postgres/Zabbix/Memcached), advanced WAF-evasion beyond the core mutators, GUI/proxy integration.

## Testability (hermetic, CI-friendly — no cloud, no internet)

Key simplifier: the metadata/target URL is an **input**, so tests point wraith at `http://127.0.0.1:<ephemeral>/…` as the "internal target." `169.254.169.254` appears only as an input string or as an address *served* by a local mock.

- **Tier 0 — Unit (hermetic, ms).** Largest tier. `pytest.parametrize` exact-byte assertions on every mutator + protocol encoder (e.g. `127.0.0.1`→`2130706433`/`0x7f000001`/`0177.0.0.1`; `169.254.169.254`→`2852039166`/`0xa9fea9fe`; `@`/`#`/`\`, `[::ffff:127.0.0.1]`, gopher-Redis byte string).
- **Tier 1 — Mocked HTTP (in-process).** `respx` (httpx) — prove the IMDSv2 PUT-then-GET handshake and header echo (assert `x-aws-ec2-metadata-token` present on the GET → proves v2 not v1). `scan-primitives` wraps `httpx`, so Tier-1 mocking uses **`respx`**.
- **Tier 2 — Local fixture servers (real loopback).** `pytest-httpserver` (client-agnostic; `expect_ordered_request` enforces AWS PUT-before-GET; header-match fails if `Metadata-Flavor` omitted). Canned fixtures under `tests/fixtures/metadata/` (aws imdsv2 token + IAM creds json, gcp token, azure instance/token). Local **OOB listener**: in-process `dnslib` DNSServer on `127.0.0.1:5353` recording QNAMEs, + httpserver for HTTP callbacks — enables the full deterministic assertion: *mutate → resolve via local resolver → hit loopback internal mock → unique-token DNS query recorded.*
- **Tier 3 — Docker (opt-in, gated marker, egress-blocked).** Flask `/fetch?url=` + `redis:alpine` on an `internal: true` network; vulhub `weblogic/ssrf`; optional `amazon-ec2-metadata-mock` sidecar. Run behind `init-firewall.sh` (CLAUDE.md R9).
- **Enforcement.** `pytest-socket --disable-socket --allow-hosts=127.0.0.1` (autouse) on Tiers 0–2 so accidental real egress fails loudly.

## Safety

- **R5 (untrusted input):** fetched response bytes are **data, never instructions** — no `eval`, no LLM-judging of fetched content.
- **Scope enforcement before ANY request** via **`scan-primitives`** (scope-file: one host/CIDR per line, `#` comments).
- Safe-by-default: dry-run for anything that could change target state; aggressive modes require explicit opt-in flags.

## Shared infra (resolved — Option C, 2026-07-02)

- **HTTP client + scope + rate-limit:** depend on the new **`scan-primitives`** shared lib (build it first — see `projects/scan-primitives/SPEC.md`). wraith benefits **most** from it: unlike doppelganger/reaper, *all* of wraith's traffic is well-formed HTTP the shared client can carry.
- **Findings:** implement a per-tool `Finding` to the **pinned contract** in `scan-primitives/SPEC.md` (lowercase severity, `cwe_id=918`); export SARIF 2.1.0; adopt **`h1-reporter`** for HackerOne markdown. A dedicated `web-finding-schema` lib is deferred; the pinned shape makes later extraction a move, not a rewrite.
- **Mocking:** `scan-primitives` wraps `httpx` → Tier-1 tests use `respx`.
